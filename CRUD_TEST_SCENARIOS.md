# Сценарии тестирования CRUD для таблицы `users`

Ниже приведены сценарии тестирования для таблицы `users` в банковской системе.

## Таблица

```sql
users(
    id BIGINT PRIMARY KEY,
    full_name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    status VARCHAR(20) NOT NULL
)
```

Допустимые значения поля `status`:

- `ACTIVE`
- `BLOCKED`

## 1. Create

### Позитивный сценарий

- Действие: добавить нового пользователя с корректными данными.
- Пример данных:
  - `id = 1`
  - `full_name = Ivan Petrov`
  - `email = ivan.petrov@bank.local`
  - `status = ACTIVE`
- Ожидаемый результат:
  - запись успешно добавляется;
  - количество записей в таблице увеличивается на 1;
  - данные нового пользователя корректно сохраняются в БД.

### Негативный сценарий

- Действие: добавить пользователя с уже существующим `email`.
- Ожидаемый результат:
  - операция завершается ошибкой ограничения уникальности;
  - новая запись не создаётся;
  - существующие данные не изменяются.

## 2. Read

### Позитивный сценарий

- Действие: получить список всех пользователей.
- Ожидаемый результат:
  - запрос успешно возвращает список пользователей;
  - каждая запись содержит корректные поля `id`, `full_name`, `email`, `status`;
  - данные соответствуют ранее сохранённым значениям.

### Негативный сценарий

- Действие: получить пользователя по несуществующему `id`.
- Ожидаемый результат:
  - запрос выполняется без ошибки;
  - результат пустой;
  - лишние или случайные записи не возвращаются.

## 3. Update

### Позитивный сценарий

- Действие: обновить статус пользователя с `ACTIVE` на `BLOCKED`.
- Ожидаемый результат:
  - обновляется ровно одна запись;
  - новое значение статуса сохраняется в таблице;
  - остальные поля записи не меняются.

### Негативный сценарий

- Действие: попытаться обновить статус на некорректное значение, например `UNKNOWN`.
- Ожидаемый результат:
  - операция завершается ошибкой ограничения;
  - некорректное значение не сохраняется;
  - предыдущее корректное значение остаётся без изменений.

## 4. Delete

### Позитивный сценарий

- Действие: удалить пользователя по существующему `id`.
- Ожидаемый результат:
  - запись удаляется из таблицы;
  - количество записей уменьшается на 1;
  - удалённый пользователь больше не возвращается при чтении.

### Негативный сценарий

- Действие: удалить пользователя по несуществующему `id`.
- Ожидаемый результат:
  - ошибка не возникает;
  - удаляется 0 записей;
  - существующие данные остаются без изменений.
