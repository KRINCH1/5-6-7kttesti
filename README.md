# Самостоятельные задания по теме "Банк"

Проект в папке `D:\bank-kt` полностью независим от дипломной работы и содержит отдельные контрольные точки:

- `КТ-5` - тестирование резервного копирования и восстановления данных;
- `КТ-6` - тестирование миграций и версий базы данных;
- `КТ-7` - тестирование безопасности базы данных.

## Используемые технологии

- Java 17
- Maven
- H2 Database
- Liquibase
- JUnit 5

## Структура проекта

- `src/test/java/com/bank/kt/BackupRestoreTest.java` - тест для КТ-5
- `src/test/java/com/bank/kt/MigrationVersioningTest.java` - тест для КТ-6
- `src/test/java/com/bank/kt/DatabaseSecurityTest.java` - тест для КТ-7
- `src/test/resources/db/changelog` - миграции Liquibase для сценария банка
- `maven-settings.xml` - локальная настройка Maven для автономной проверки проекта

## Как проверить КТ-5, КТ-6 и КТ-7

### 1. Открыть проект

Откройте папку проекта:

```powershell
cd D:\bank-kt
```

### 2. Запустить все контрольные точки сразу

```powershell
mvn -s maven-settings.xml test
```

Если всё выполнено успешно, в конце будет результат:

```text
Tests run: 5, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

## Как проверить каждую контрольную точку отдельно

### КТ-5. Тестирование резервного копирования и восстановления данных

Запуск:

```powershell
mvn -s maven-settings.xml -Dtest=BackupRestoreTest test
```

Что проверяется:

- создаётся тестовая база банка;
- выполняется резервное копирование через стандартную команду H2 `SCRIPT`;
- выполняется восстановление через `RUNSCRIPT`;
- проверяется, что таблицы `customers`, `accounts`, `transactions` и все записи восстановлены без потерь.

### КТ-6. Тестирование миграций и версий базы данных

Запуск:

```powershell
mvn -s maven-settings.xml -Dtest=MigrationVersioningTest test
```

Что проверяется:

- применение миграций Liquibase;
- добавление новой таблицы `branches`;
- добавление нового столбца `loyalty_level` в таблицу `customers`;
- удаление столбца `legacy_code` из таблицы `accounts`;
- сохранность данных после изменения схемы.

### КТ-7. Тестирование безопасности базы данных

Запуск:

```powershell
mvn -s maven-settings.xml -Dtest=DatabaseSecurityTest test
```

Что проверяется:

- создание пользователей `bank_admin`, `bank_user`, `bank_guest` с разными правами;
- запрет для пользователя `guest` на удаление записей;
- защита от SQL-инъекций с помощью `PreparedStatement`;
- хранение конфиденциальных данных в зашифрованном виде.

## Итог

Проект реализован как самостоятельное учебное задание по теме `банк` и не связан с дипломной работой. Все три контрольные точки можно проверить как вместе, так и по отдельности через Maven-команды выше.
